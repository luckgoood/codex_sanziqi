# 发现与决策

## 需求

- 在终端游戏启动时提供菜单选择。
- 支持玩家对电脑与双人对战。
- 保持 Fortran + PowerShell 构建方式。
- 按项目要求在修改后提交并推送到 GitHub。

## 项目发现

- 主程序为 `src/main.f90`，使用内部子程序组织逻辑。
- 棋盘元素是 `character(len=3)`，以兼容 UTF-8 的 `△`。
- 现有电脑策略依次尝试获胜、阻挡玩家、按固定优先级选择空位。
- `build.ps1` 会使用 `gfortran` 编译到 `build\\tic_tac_toe.exe`。

## 技术决策

| 决定 | 原因 |
|---|---|
| 用整数 `game_mode` 保存菜单选择 | 与现有 `read`/`iostat` 输入校验方式一致。 |
| 双人模式复用通用 `human_turn` | 同时校验数字、范围和占用位置。 |
| 游戏结束文案由模式和胜者共同决定 | 电脑模式显示“你/电脑”，双人模式显示“玩家 1/玩家 2”。 |

## 已遇问题

| 问题 | 处理方式 |
|---|---|
| 本地 skill 的恢复检查与多项读取并行调用未返回可用结果 | 单独读取模板和项目文件继续执行。 |
| 测试进程找不到 gfortran 运行库，退出码为 `-1073741515` | 临时把 `D:\\msys64\\ucrt64\\bin` 加入测试进程的 `Path`。用户正常运行时也需要该目录在 `Path` 中。 |
| PowerShell 标准输入测试未在预期步数结束 | 改用临时输入文件配合 `cmd.exe` 重定向，两个模式均已完成验证。 |
